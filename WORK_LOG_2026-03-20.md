## 서비스 부팅 시 자동 시작 설정

### 개요
영어 교육 플랫폼의 백엔드와 프론트엔드 서비스가 시스템 부팅 시 자동으로 시작되도록 systemd 서비스를 구성했습니다.

### 생성된 서비스 파일

#### 1. Backend Service
**파일 위치**: `/etc/systemd/system/english-fairy-backend.service`

```ini
[Unit]
Description=English Fairy Backend API
After=network.target docker.service
Wants=docker.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/english-learning-platform/backend
Environment="PATH=/home/ubuntu/english-learning-platform/backend/.venv/bin:/usr/local/bin:/usr/bin:/bin"
ExecStart=/home/ubuntu/english-learning-platform/backend/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

**특징**:
- Docker 서비스 이후에 시작
- 포트 8000에서 실행
- 실패 시 10초 후 자동 재시작
- Python 가상환경 사용

#### 2. Frontend Service
**파일 위치**: `/etc/systemd/system/english-fairy-frontend.service`

```ini
[Unit]
Description=English Fairy Frontend
After=network.target english-fairy-backend.service
Wants=english-fairy-backend.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/english-learning-platform/frontend
Environment="PATH=/usr/local/bin:/usr/bin:/bin"
Environment="NODE_ENV=development"
ExecStart=/usr/bin/npm run dev -- --hostname 0.0.0.0
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

**특징**:
- 백엔드 서비스 이후에 시작
- 포트 3000에서 실행 (Next.js 개발 서버)
- 실패 시 10초 후 자동 재시작
- 모든 네트워크 인터페이스에서 접근 가능 (0.0.0.0)

### 설정 명령어

```bash
# 서비스 파일 복사
sudo cp /home/ubuntu/english-fairy-backend.service /etc/systemd/system/
sudo cp /home/ubuntu/english-fairy-frontend.service /etc/systemd/system/

# systemd 데몬 리로드
sudo systemctl daemon-reload

# 부팅 시 자동 시작 활성화
sudo systemctl enable english-fairy-backend
sudo systemctl enable english-fairy-frontend

# 서비스 시작
sudo systemctl start english-fairy-backend
sudo systemctl start english-fairy-frontend
```

### 유용한 관리 명령어

```bash
# 서비스 상태 확인
sudo systemctl status english-fairy-backend
sudo systemctl status english-fairy-frontend

# 로그 실시간 확인
```

## 알파벳 레슨 버그 수정

### 문제 상황
"오늘의 미션 - 알파벳 A부터 Z까지" 레슨을 실행하면 "글자 데이터가 없습니다" 오류가 발생했습니다.

### 원인 분석

#### 1. 데이터베이스 구조 확인
```sql
-- 알파벳 레슨 데이터 확인
SELECT id, lesson_id, content_data
FROM lesson_items
WHERE lesson_id = 'fdfc9e2f-db73-41f3-8fbc-8059e79022ca'
LIMIT 5;
```

**결과**: 데이터베이스에는 데이터가 정상적으로 존재
```json
{
  "letter": "A",
  "upper": "A",
  "lower": "a",
  "keyword": "apple",
  "sound": "æ"
}
```

#### 2. 코드 분석
프론트엔드 코드(`/frontend/src/app/(child)/learn/phonics/[lessonId]/page.tsx`)가 다음 필드를 찾고 있었음:
- `word` (문자열)
- `phonemes` (배열)

하지만 알파벳 레슨의 데이터 구조는:
- `letter` (문자)
- `upper`, `lower` (대소문자)
- `keyword` (키워드)
- `sound` (발음)

#### 3. 두 가지 레슨 타입 발견
- **알파벳 레슨**: `content_type: "letter_sound"` - 개별 글자 학습
- **파닉스 블렌딩 레슨**: `content_type: "phonics_word"` - 소리 합치기 학습

### 수정 내용

**파일**: `/home/ubuntu/english-learning-platform/frontend/src/app/(child)/learn/phonics/[lessonId]/page.tsx`

**수정 전** (48-67번 줄):
```typescript
// Parse lesson items into phonics words
const words: PhonicsItem[] = (lesson?.items ?? []).map((item, idx) => {
  const word = typeof item.content_data?.word === "string" ? item.content_data.word : "";
  const phonemes = Array.isArray(item.content_data?.phonemes) ? item.content_data.phonemes : [];

  return {
    word,
    phonemes,
  };
});
```

**수정 후**:
```typescript
// Parse lesson items into phonics words
const words: PhonicsItem[] = (lesson?.items ?? []).map((item, idx) => {
  // Handle different content types
  let word = "";
  let phonemes: string[] = [];

  if (item.content_type === "letter_sound") {
    // Alphabet lesson: use letter as both word and phoneme
    word = typeof item.content_data?.letter === "string" ? item.content_data.letter : "";
    phonemes = word ? [word] : [];
  } else if (item.content_type === "phonics_word") {
    // Phonics blending lesson
    word = typeof item.content_data?.word === "string" ? item.content_data.word : "";
    phonemes = Array.isArray(item.content_data?.phonemes) ? item.content_data.phonemes : [];
  }

  return {
    word,
    phonemes,
  };
});
```
